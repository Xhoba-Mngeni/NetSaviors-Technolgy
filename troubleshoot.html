Follow these standardized steps to efficiently resolve technical issues

Step 1: Information Gathering

  • "Can you describe what you were trying to do when the problem occurred?"
  • "What is the exact error message you're seeing?"
  • "Which specific application or system are you using?"
  • "What were you doing right before the problem started?"

Step 2: Identify Symptoms

  • "When did you first notice this issue?"
  • "Is the problem consistent or intermittent?"
  • "Does it affect only you or multiple users?"
  • "Can you reproduce the issue now?"

Step 3: Establish Theory

  • "Have there been any recent changes to your system?"
  • "Did you install any new software recently?"
  • "Has there been a power outage or network disruption?"
  • "Have you tried any solutions already?"

Step 4: Test Theory

  • Ask user to restart the application
  • Have user restart their computer
  • Test on different user profile if possible
  • Check network connectivity and permissions
  • Verify software updates are current

Step 5: Resolution & Verification

  • "Can you confirm the issue is now resolved?"
  • "Please test all related functions"
  • Ensure no secondary issues were created
  • Document the solution for future reference
  • Update knowledge base with resolution
